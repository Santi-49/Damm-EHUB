# Challenge

## Hackathon

- **Name:** LineWise Operaciones — Damm × Engineering HUB
- **Organizer:** Damm (canning plant, El Prat de Llobregat) + Engineering HUB
- **Lines in scope:** L14, L17, L19 (can-filling lines)
- **Deadline / Demo time:** TBD (set by organizer)
- **Submission format:** GitHub repo + working demo + IA component + short pitch

## Problem Statement

Damm canning lines change between SKUs many times per week. Every changeover costs setup
time and reduces OEE (Overall Equipment Effectiveness). The plant currently runs a
heuristic plan generated by Blue Yonder / JDA, but it does not always match what
operators actually execute. The challenge asks for a tool that:

1. **Detects historical changes that hurt OEE** and explains *why* they hurt.
2. **Predicts OEE impact** of a new sequence, changeover, or line reassignment.
3. **Compares scenarios** (original plan vs alternative).
4. **Recommends line + sequence** with estimated OEE impact, including under urgent
   demand or breakdowns.
5. **Visualises the schedule** and lets the user move slots interactively to see
   the impact (Gantt drag-and-drop is explicit in the brief).
6. Ships as a working demo + repo with a real **IA component** (not just a dashboard).

## Domain / Industry

Beer canning operations. Three filling lines with format restrictions:

| Line | Allowed formats |
|---|---|
| **L14** | 1/2 (50 cl) and 1/3 (33 cl) |
| **L17** | 1/3 (33 cl) only |
| **L19** | 1/2 (50 cl), 1/3 (33 cl), 2/5 (44 cl) |

These are **hard constraints** — a SKU cannot be planned on a line whose format it does
not support. Quality is treated as Q ≡ 1 throughout 2025 (Damm reports OEE = Availability × Performance).

## Mandatory Technologies

- **IA component is mandatory** — must be more than a lookup or rule engine.
- Streamlit / dashboarding / generative AI / AutoML are all allowed (FAQ §8).
- No restriction on the optimisation stack.

## Rules & Hard Constraints

- **Data is confidential** — must not leave the hackathon. `.gitignore` blocks all
  `data/` subfolders and `*.xlsx`.
- **The demo must run end-to-end** (not a static maquette).
- **The Gantt must be interactive** (move elements to see impact) — explicit in §3 / §5.
- **Document assumptions** where data is ambiguous (see `docs/linewise/datos.md` §5).
- **External data sources allowed and welcome** with citation — we decided not to use
  any (rationale in `docs/linewise/cobertura_brief.md` §6).
