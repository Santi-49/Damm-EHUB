## Request Flow

```
Client
  │
  │  1. POST /api/v1/auth/login  →  returns access_token + refresh_token
  │                                  (both stored in Redis whitelist)
  │
  │  2. GET  /api/v1/some/route
  │     Authorization: Bearer <access_token>
  │
  ▼
FastAPI middleware
  │
  ├─ 3. Decode JWT (python-jose) — verify signature + expiry
  │
  ├─ 4. Redis GET token:{jti}  — must exist (whitelist = revocation guard)
  │
  ├─ 5. Load User from Postgres via sub (user_id)
  │
  └─ 6. Call OPA:  POST /v1/data/authz/allow
         input: { user_id, roles, resource, action }
         result: { allow: true | false }

Route handler
  │
  └─ 7. Call services/  →  module contract (challenge logic)
```

## Contract Boundaries

Two explicit seams allow parallel development during a hackathon:

| Boundary | Mechanism | Reference |
|---|---|---|
| Frontend ↔ Backend | OpenAPI spec (auto-generated by FastAPI) → TypeScript types | `packages/contracts/api/` |
| Backend ↔ Module | Python `Protocol` class | `packages/contracts/module/interface.py` |

Run `make generate-types` to regenerate TypeScript types from the live OpenAPI spec.

See [Module Contracts](../contracts.md) for the full workflow.

## Docker Compose Services

| Service | Image | Internal port | Dev exposed port |
|---|---|---|---|
| `postgres` | postgres:16-alpine | 5432 | 5432 |
| `redis` | redis:7-alpine | 6379 | 6379 |
| `opa` | openpolicyagent/opa:latest-rootless | 8181 | 8181 |
| `api` | (built from `services/api`) | 8000 | 8000 |

All services declare health checks. The `api` service waits for all three dependencies to be healthy before starting.
